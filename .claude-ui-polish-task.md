## Task 2: Fix Lyrics Auto-Scroll

The KaraokeLyrics component auto-scrolls but it's janky. The issues:
- Uses `scrollTo` with `behavior: "smooth"` which is fine, but the scroll check runs on every `currentTimeMs` change
- The container has `mask-image` gradient that creates fade at top/bottom — good, keep it

**Fix in `apps/web/src/components/KaraokeLyrics.tsx`**:
- Throttle scroll checks: only check every 500ms, not every frame
- Use `scrollIntoView({ block: 'center', behavior: 'smooth' })` on the active line ref instead of manual offset calculation
- When user manually scrolls, pause auto-scroll for 5 seconds (not 3)

Current scroll logic (around line 50-67):
```tsx
// Replace the useEffect for scrolling with:
useEffect(() => {
  if (!autoScroll || !activeRef.current) return;
  activeRef.current.scrollIntoView({ block: 'center', behavior: 'smooth' });
}, [autoScroll]);
```

And add a separate effect that tracks when the active line changes:
```tsx
// Track active line change
const prevActiveRef = useRef<number>(-1);
const activeIndex = useMemo(() => { /* existing logic */ }, [lines, currentTimeMs]);

useEffect(() => {
  if (activeIndex !== prevActiveRef.current) {
    prevActiveRef.current = activeIndex;
    if (autoScroll && activeRef.current) {
      activeRef.current.scrollIntoView({ block: 'center', behavior: 'smooth' });
    }
  }
}, [activeIndex, autoScroll]);
```

Also increase the user scroll resume delay from 3000ms to 5000ms.
