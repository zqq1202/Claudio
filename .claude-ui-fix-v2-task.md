## Bug 1: 歌词自动滚动完全失效

**文件**: `apps/web/src/components/KaraokeLyrics.tsx`

**根因**: `scrollIntoView({ behavior: 'smooth' })` 会改变滚动位置 → 触发 `onScroll` 事件 → `handleScroll` 设 `autoScroll = false` → 后续的 scrollIntoView 调用被阻止。也就是说 scrollIntoView 自己触发了 onScroll，把自己给禁用了。

**修复**: 用一个 `isScrollingRef` 标志位来区分程序触发的滚动和用户手动滚动：

```tsx
// 添加 ref
const isScrollingRef = useRef(false);

// 修改 scrollIntoView 调用（第 107-117 行的 useEffect）
useEffect(() => {
  if (activeIndex !== prevActiveRef.current) {
    prevActiveRef.current = activeIndex;
    if (autoScroll && activeLineRef.current) {
      isScrollingRef.current = true;  // 标记为程序滚动
      activeLineRef.current.scrollIntoView({
        block: 'center',
        behavior: 'smooth',
      });
      // 300ms 后清除标记（smooth scroll 动画完成）
      setTimeout(() => { isScrollingRef.current = false; }, 300);
    }
  }
}, [activeIndex, autoScroll]);

// 修改 handleScroll
const handleScroll = useCallback(() => {
  if (isScrollingRef.current) return;  // 忽略程序触发的滚动
  if (userScrollTimer.current) clearTimeout(userScrollTimer.current);
  setAutoScroll(false);
  userScrollTimer.current = setTimeout(() => setAutoScroll(true), 5000);
}, []);
```

另外，将 scrollIntoView 的容器改为 `.karaoke-panel`（当前是 `activeLineRef.current`，但 activeLineRef 可能指向一个不在 DOM 中的元素因为 React re-render 时序问题）。用 panel ref 做 scrollTo 更可靠：

实际上保持 scrollIntoView 就行，只要修复 onScroll 冲突问题。

## Bug 3: 底部频谱条抖动非常不自然

**文件**: `apps/web/src/components/SpectrumBars.tsx`

**根因**: release 系数 0.92 太大（每帧消除 92% 的差距 = 柱子几乎瞬间掉落）。对比 AudioVisualizer 使用 RELEASE=0.3（慢衰减）。

**修复** — 修改整个动画参数体系，使其更接近 AudioVisualizer 的自然感：

1. **smoothingTimeConstant**: 改为 `0.75`（更多 FFT 平滑）
```ts
analyser.smoothingTimeConstant = 0.75;
```

2. **Attack/Release 系数**:
```ts
// Fast attack (0.45), VERY slow release (0.08) — 弹起快，缓慢下降
bars[i] += (target - bars[i]) * (target > bars[i] ? 0.45 : 0.08);
```

3. **动态范围压缩** — sqrt 指数改为 0.6（更强压缩，避免低频过高）:
```ts
const barHeight = Math.pow(bass[i], 0.6) * h * 0.9;
```

4. **颜色渐变** — 底端更亮:
```ts
grad.addColorStop(0, toRgba(primaryColor, 0.08));
grad.addColorStop(0.3, toRgba(primaryColor, 0.25));
grad.addColorStop(0.6, toRgba(primaryColor, 0.55));
grad.addColorStop(1, toRgba(primaryColor, 0.95));
```

5. **增加低频 boost** — 让低频更突出:

## Bug 4: AI 歌曲封面始终加载失败

**文件**: `apps/server/src/routes/dispatch.ts`, `apps/server/src/routes/cover.ts`, `apps/server/src/services/ncm.service.ts`

虽然 dispatch.ts 已经始终用 NCM coverUrl 覆盖，但有以下可能原因：
1. NCM search 搜索 `artist name` 格式可能匹配不到（搜索质量差）
2. 即使匹配到了，`mapSong` 可能取不到 `picUrl`（NCM API 返回格式变化）
3. `/api/cover` proxy 在 fetch 时没有带 Cookie，某些图片需要认证

**多重修复**:

### Fix A: cover proxy 带上 NCM Cookie
**文件**: `apps/server/src/routes/cover.ts`

在 fetch 时加上 Cookie header（从环境变量或配置读取）:
```ts
// 读取 NCM cookie（从 settings 或环境变量）
const ncmCookie = (app as any).services?.context?.getSetting?.("ncm_cookie") ?? process.env.NCM_COOKIE ?? "";

const res = await fetch(url, {
  headers: {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36",
    Referer: "https://music.163.com/",
    ...(ncmCookie ? { Cookie: ncmCookie } : {}),
  },
});
```

但 cover.ts 的路由注册没有访问 app.services。一个更简单的方法：直接从 config 读取 cookie，或者在路由注册时通过闭包传入。

**最简单方案**: cover.ts 从环境变量读 cookie:
```ts
const NCM_COOKIE = process.env.NCM_COOKIE ?? "";

const res = await fetch(url, {
  headers: {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36",
    Referer: "https://music.163.com/",
    ...(NCM_COOKIE ? { Cookie: NCM_COOKIE } : {}),
  },
});
```

### Fix B: 增强 dispatch.ts 的歌曲搜索和封面获取
**文件**: `apps/server/src/routes/dispatch.ts`

搜索失败时尝试只用歌名搜索，并确保 cover 始终有值:

### Fix C: 客户端 SongCard fallback
**文件**: `apps/web/src/components/SongCard.tsx`

如果 coverUrl 为空但有 songId，尝试通过 NCM 歌曲 ID 获取封面:
```tsx
// 在 SongCard 组件顶部添加 fallback URL
const effectiveCoverUrl = coverUrl || (songId ? `/api/cover?url=${encodeURIComponent(`https://p1.music.126.net/song/media/outer/url?id=${songId}.jpg`)}` : null);
```

等等，NCM 没有这样的 URL。更实际的方案：直接显示占位符但用更好的样式。

实际上最有效的方案是：**在 SongCard 和 LegacySongCard 中，如果 coverUrl 存在但加载失败（onError），尝试通过后端获取封面**:

```tsx
<img 
  src={coverUrl} 
  alt={title} 
  loading="lazy"
  onError={(e) => {
    // Fallback: 通过后端搜索获取封面
    const img = e.currentTarget;
    if (songId && !img.dataset.retried) {
      img.dataset.retried = "1";
      fetch(`/api/song/cover?id=${songId}`)
        .then(r => r.ok ? r.json() : null)
        .then(data => {
          if (data?.coverUrl) img.src = data.coverUrl;
        })
        .catch(() => {});
    }
  }}
/>
```

然后在 server 端添加 `GET /api/song/cover?id=xxx` 端点，调用 NCM 获取歌曲详情和封面。

### Fix D: 新增 /api/song/cover 端点

```ts
app.get("/api/song/cover", async (request, reply) => {
  const { id } = request.query as { id: string };
  if (!id) return reply.code(400).send({ error: "missing id" });

});
```

## Bug 5: DJ 语音应使用 Fish Audio TTS 而非浏览器 TTS

**当前**: `apps/web/src/utils/voiceSynth.ts` 使用浏览器 `SpeechSynthesis` API，音质差。
**目标**: 调用后端 Fish Audio API 生成高质量语音。

### Step 1: 添加服务端 TTS 端点
**文件**: 新增 `apps/server/src/routes/tts.ts` 或在现有路由中添加

```ts
app.post<{ Body: { text: string } }>("/api/tts", async (request, reply) => {
  const { text } = request.body as { text: string };
  if (!text || text.length > 500) {
    return reply.code(400).send({ error: "invalid text" });
  }
  try {
    const audioUrl = await tts.synthesize(text);
    if (!audioUrl) {
      return reply.code(503).send({ error: "TTS service unavailable" });
    }
    return { audioUrl };
  } catch (err) {
    return reply.code(500).send({ error: "TTS synthesis failed" });
  }
});
```

然后在 `apps/server/src/index.ts` 中注册这个路由（如果新建文件的话）。

### Step 2: 修改客户端 voiceSynth.ts
**文件**: `apps/web/src/utils/voiceSynth.ts`

```ts
import { audioPlayer } from "../audio/AudioPlayer";

let speaking = false;
let currentAudio: HTMLAudioElement | null = null;

export async function speak(text: string): Promise<void> {
  if (speaking) return;
  speaking = true;

  // Duck music volume
  audioPlayer.duck(0.2, 300);
  window.dispatchEvent(new CustomEvent("voiceStart", { detail: { text } }));

  try {
    // Call server TTS endpoint (Fish Audio)
    const res = await fetch("/api/tts", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ text }),
    });

    if (!res.ok) {
      throw new Error(`TTS failed: ${res.status}`);
    }

    const { audioUrl } = await res.json();
    if (!audioUrl) throw new Error("No audio URL");

    // Play the TTS audio
    await playAudio(audioUrl);
  } catch (err) {
    console.warn("[voiceSynth] Fish Audio TTS failed, falling back to browser TTS:", err);
    // Fallback to browser SpeechSynthesis
    await browserSpeak(text);
  } finally {
    speaking = false;
    audioPlayer.unduck(600);
    window.dispatchEvent(new CustomEvent("voiceEnd"));
  }
}

function playAudio(url: string): Promise<void> {
  return new Promise((resolve, reject) => {
    const audio = new Audio(url);
    currentAudio = audio;
    audio.onended = () => { currentAudio = null; resolve(); };
    audio.onerror = () => { currentAudio = null; reject(new Error("Audio play failed")); };
    audio.play().catch(reject);
  });
}

function browserSpeak(text: string): Promise<void> {
  return new Promise((resolve) => {
    const synth = window.speechSynthesis;
    if (!synth) { resolve(); return; }
    const utterance = new SpeechSynthesisUtterance(text);
    utterance.lang = "zh-CN";
    utterance.rate = 0.95;
    utterance.pitch = 1;
    utterance.onend = () => resolve();
    utterance.onerror = () => resolve();
    synth.speak(utterance);
  });
}

export function stop(): void {
  if (currentAudio) {
    currentAudio.pause();
    currentAudio = null;
  }
  window.speechSynthesis?.cancel();
  speaking = false;
  audioPlayer.unduck(600);
  window.dispatchEvent(new CustomEvent("voiceEnd"));
}

export function isSpeaking(): boolean {
  return speaking;
}
```

注意: `speak` 函数签名从 `void` 变为 `Promise<void>`。需要在 chatStore.ts 中把 `speak(structuredReply.segue)` 改为 `speak(structuredReply.segue).catch(() => {})` 。
