# Claudio UI Redesign — Design Document

> Inspired by [MoeKoeMusic](https://github.com/MoeKoeMusic/MoeKoeMusic), adapted for Claudio's dark-first cyber radio aesthetic.

## 1. Design Goals

1. **Fix all light-theme bugs** — top nav, BorderGlow, fluid blobs, noise overlay
2. **Remove the ugly green glow frame** — BorderGlow is too aggressive, looks like a bug
3. **Refine the dark theme** — polish spacing, shadows, and glassmorphism
4. **Improve light theme** — cohesive, not just "inverted dark"
5. **Borrow MoeKoeMusic patterns** — card elevation, consistent radius scale, pill tags, clean navigation

## 2. MoeKoeMusic Design Patterns to Adopt

### What MoeKoeMusic does well (and Claudio should learn from):
- **Consistent radius scale**: 8px cards, 10px content, 999px pills — not random values
- **Clean shadows**: soft, layered (`0 4px 12px rgba(...)`) not hard outlines
- **Glassmorphism done right**: `backdrop-filter: blur()` with `rgba` backgrounds, not opaque cards
- **Navigation**: clean top bar with active state indicators (colored text + underline, not just opacity)
- **Card hover states**: subtle `transform: scale(1.02)` + shadow lift
- **Spacing rhythm**: 8px base unit, multiples of 8 for sections (16, 24, 32, 48)
- **Color system**: CSS custom properties with light/dark overrides per variable

### What to NOT copy:
- MoeKoeMusic's pink/cute aesthetic — Claudio is dark/cyber/neon
- Light-white backgrounds — Claudio's light theme should be soft gray, not pure white
- Anime character decorations — not part of Claudio's identity

## 3. Bug Fixes (Priority: Critical)

### Bug 1: Top Nav Bar Doesn't Change in Light Theme
**File:** `apps/web/src/styles/global.css`
- `.top-nav` has hardcoded `background: rgba(11, 11, 16, 0.8)`
- **Fix:** Add `--bg-nav` CSS variable, override in `[data-theme="light"]`

### Bug 2: BorderGlow Creates Visible Green Frame
**File:** `apps/web/src/components/BorderGlow.tsx`
- 4 glow layers with alpha 0.08/0.15/0.25/0.35 = very visible green frame
- **Fix:** Reduce all alpha values by ~60%, reduce wave amplitude from 10→5

### Bug 3: Light Theme Missing Many Overrides
**File:** `apps/web/src/styles/global.css`
- Fluid blobs, noise overlay, player card backgrounds, chat section, etc.
- **Fix:** Add comprehensive `[data-theme="light"]` overrides

## 5. Component-Specific Fixes

### 5.1 Top Nav Bar (`.top-nav`)
- Use `background: var(--bg-nav)` instead of hardcoded value
- Light mode: soft gray glassmorphism, not dark
- Active tab: add bottom border accent `border-bottom: 2px solid var(--mint)`

### 5.2 BorderGlow Component
- **Dramatically reduce intensity** — should be an ambient hint, not a visible frame
- Alpha layers: 0.03, 0.06, 0.10, 0.15 (down from 0.08, 0.15, 0.25, 0.35)
- Wave amplitude: 5 (down from 10)
- Light mode: set canvas opacity to 0.3

### 5.3 Fluid Blobs
- Light mode: `mix-blend-mode: multiply`, reduced opacity (0.25 primary, 0.2 secondary)
- Dark mode: keep as-is (already looks good)

### 5.4 Noise Overlay
- Light mode: reduce `body::before` opacity from 0.5 to 0.2
- The noise texture is too visible on light backgrounds

### 5.5 Player Card
- Add subtle shadow: `var(--shadow-player-card)`
- Light mode: white card with soft shadow, not glassmorphism

### 5.6 Chat Section
- Light mode: white background, soft border
- Light mode chat messages: subtle gray backgrounds

### 5.7 Controls (buttons, sliders)
- Hover states: add scale(1.05) + shadow lift (MoeKoeMusic pattern)
- Active states: mint color tint
- Light mode: controls should use dark icons on light background

### 5.8 Search Panel / Extras Panel
- Light mode: white cards with soft shadows
- Theme buttons: use active state with mint accent
