## Bug 1: 歌词自动滚动失效

**文件**: `apps/web/src/components/KaraokeLyrics.tsx`

**问题**: `scrollIntoView({ behavior: 'smooth' })` 会持续触发 `onScroll` 事件。虽然设置了 `isScrollingRef.current = true` 300ms，但 smooth scroll 可能超过 300ms，导致中间的 scroll 事件触发 `handleScroll` → `setAutoScroll(false)`，自动滚动被禁用。

**修复方案**: 改用 `scrollTo` + 计算精确滚动位置，不用 `scrollIntoView`（它不可控）。同时用一个 `isProgrammaticScroll` ref 标志位：

```tsx
// 1. 添加 containerRef 回来
const containerRef = useRef<HTMLDivElement>(null);

// 2. 替换 scrollIntoView 为 scrollTo
useEffect(() => {
  if (activeIndex !== prevActiveRef.current) {
    prevActiveRef.current = activeIndex;
    if (autoScroll && containerRef.current && activeLineRef.current) {
      const container = containerRef.current;
      const line = activeLineRef.current;
      const containerH = container.clientHeight;
      const lineTop = line.offsetTop;
      const lineH = line.offsetHeight;
      // 滚动到容器 35% 位置
      const targetScroll = lineTop - containerH * 0.35 + lineH / 2;
      
      isScrollingRef.current = true;
      container.scrollTo({ top: targetScroll, behavior: 'smooth' });
      // smooth scroll 通常 500-800ms，给 1000ms 宽裕时间
      setTimeout(() => { isScrollingRef.current = false; }, 1000);
    }
  }
}, [activeIndex, autoScroll]);

// 3. handleScroll 保持 isScrollingRef 守卫
const handleScroll = useCallback(() => {
  if (isScrollingRef.current) return;
  if (userScrollTimer.current) clearTimeout(userScrollTimer.current);
  setAutoScroll(false);
  userScrollTimer.current = setTimeout(() => setAutoScroll(true), 8000);
}, []);

// 4. JSX 中加回 containerRef
<div className="karaoke-panel" ref={containerRef} onScroll={handleScroll}>
```

**关键**: 不要用 `scrollIntoView`，改用 `scrollTo` + 手动计算 `lineTop - containerH * 0.35 + lineH / 2`。timeout 改为 1000ms。用户手动滚动恢复时间改为 8000ms。

## Bug 2: 歌词高亮应使用主题色

**文件**: `apps/web/src/styles/global.css`

**问题**: `.karaoke-char.lit` 和 `.karaoke-sweep` 的 text-shadow 中仍有硬编码的 `rgba(94,232,197,0.2)`，不会跟随主题色变化。应全部使用 `var(--color-primary)` 的派生色。

```css
.karaoke-char.lit {

  text-shadow: 0 0 20px var(--color-primary), 0 0 40px color-mix(in srgb, var(--color-primary) 30%, transparent);

/* 活跃行的 container 发光也用主题色 */
.karaoke-line-wrapper.active {

}
```

对 `.karaoke-sweep` 的 CSS 渐变也一样确认使用 `var(--color-primary)` 而非硬编码。

## Bug 3: 底部频谱条不自然

**文件**: `apps/web/src/components/SpectrumBars.tsx`

**问题**: 当前 release 系数 0.08 太小（柱子几乎不动），attack 0.45 也偏小。用户说"前面一半段完全没怎么动过，后面一半段就稍微动一动"。对比上面的 AudioVisualizer，它用 `ATTACK=0.6, RELEASE=0.3, smoothingTimeConstant=0.8`，效果好很多。

**修复**:

```tsx
// analyser smoothing 调高
analyser.smoothingTimeConstant = 0.8;  // 原 0.75

// 攻击/释放比改为类似 AudioVisualizer
// attack: 快速响应（0.65），release: 中速回落（0.25）
bars[i] += (target - bars[i]) * (target > bars[i] ? 0.65 : 0.25);

// bass boost 保持但略微降低，避免过度放大
const boost = 1.0 + Math.pow(1.0 - ratio, 2.0) * 1.5;  // 原 2.5 * 2.0 过强

// 柱子高度缩放稍微增加
const barHeight = Math.pow(bars[i], 0.65) * h * 0.88;  // 原 0.6 * 0.9

// glow 适度
ctx.shadowBlur = 4 + bars[i] * 10;  // 原 6 + bars[i] * 12
```

核心改动：`smoothingTimeConstant` 0.75→0.8，attack 0.45→0.65，release 0.08→0.25。这样柱子会跟随音乐律动自然弹跳。

## Bug 4: AI 推荐歌曲封面仍然加载失败

**文件**: 
- `apps/server/src/routes/dispatch.ts` — `/api/song/cover` 端点有 bug
- `apps/server/src/services/ncm.service.ts` — 缺少 `getSongDetail` 方法
- `apps/server/ncm-server.mjs` — 缺少 `/song/detail` 端点

**根因**: `/api/song/cover?id=xxx` 调用 `ncm.search(id, 1)`，但 `id` 是数字歌曲 ID，不是关键词！`search` 会用 ID 号当关键词搜索，肯定找不到。

**修复方案**:

### A. ncm-server.mjs — 添加 `/song/detail` 端点

在 ncm-server.mjs 中现有的 `/recommend/songs` 路由之前（约 line 200），添加：

```js
} else if (url.pathname === "/song/detail") {
  const ids = url.searchParams.get("ids") ?? "";
  const idList = ids.split(",").filter(Boolean);
  if (idList.length === 0) {
    res.statusCode = 400;
    res.end(JSON.stringify({ error: "missing ids" }));
    return;
  }
  const data = await ncmPost("/weapi/v3/song/detail", {
    c: JSON.stringify(idList.map(id => ({ id }))),
  });
  const songs = (data?.songs ?? []).map(mapSong);
  res.end(JSON.stringify({ songs }));
}
```

### B. ncm.service.ts — 添加 getSongDetail 方法

在 `NcmService` 接口中添加：
```ts
getSongDetail(songId: string): Promise<SongSearchResult | null>;
```

在 `NeteaseNcmService` 中实现：
```ts
async getSongDetail(songId: string): Promise<SongSearchResult | null> {
  try {
    const data = await this.fetchJson<any>("/song/detail", { ids: songId });
    const songs = data?.songs ?? [];
    return songs.length > 0 ? this.mapSong(songs[0]) : null;
  } catch {
    return null;
  }
}
```

在 `MockNcmService` 中添加对应 mock。

### C. dispatch.ts — 修复 `/api/song/cover` 端点

```ts
app.get<{ Querystring: { id: string } }>("/api/song/cover", async (request, reply) => {
  const { id } = request.query;
  if (!id) return reply.code(400).send({ error: "missing id" });
  try {
    const { ncm } = app.services;
    const detail = await ncm.getSongDetail(id);
    if (detail?.coverUrl) {
      return { coverUrl: detail.coverUrl };
    }
    return reply.code(404).send({ error: "not found" });
  } catch {
    return reply.code(500).send({ error: "search failed" });
  }
});
```
