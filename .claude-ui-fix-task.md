## Bug 1: AI推荐歌曲封面全部加载失败

**根因**: `apps/server/src/routes/dispatch.ts` 第 111 行:
```ts
if (!song.cover) song.cover = results[0].coverUrl;
```
AI (Claude) 生成的 `cover` 字段通常是原始网易云 URL (如 `https://p1.music.126.net/...`)，但浏览器无法直接加载（CORS 限制）。NCM search 返回的 `coverUrl` 已经是代理 URL (`/api/cover?url=...`)，但因为 `!song.cover` 条件不会被覆盖。

**修复**: 始终用 NCM search 结果的 coverUrl 替换，不管 AI 有没有提供 cover：
```ts
// dispatch.ts 第 109-112 行
if (results.length > 0) {
  song.id = results[0].id;
  song.cover = results[0].coverUrl;  // 总是覆盖，因为 NCM 返回的是代理 URL
}
```

## Bug 2: 底部音乐频谱条抖动不自然

**文件**: `apps/web/src/components/SpectrumBars.tsx`

**问题**: 
- `smoothingTimeConstant = 0.7` 过高，数据过度平滑
- 攻击系数 0.5 太慢（柱子弹起慢），释放系数 0.88 太快（柱子快速掉落）
- 颜色渐变底端太透明 (0.02 alpha)，几乎看不到
- sqrt 压缩 0.7 压缩过度

**修复**:
1. `analyser.smoothingTimeConstant = 0.6` (降低平滑，更灵敏)
2. 柱子插值: 攻击 0.65，释放 0.92 (更自然的衰减)
```ts
bars[i] += (target - bars[i]) * (target > bars[i] ? 0.65 : 0.92);
```
3. 颜色渐变更亮:
```ts
grad.addColorStop(0, toRgba(primaryColor, 0.05));
grad.addColorStop(0.3, toRgba(primaryColor, 0.15));
grad.addColorStop(0.6, toRgba(primaryColor, 0.4));
grad.addColorStop(1, toRgba(primaryColor, 0.85));
```
4. 增加 glow:
```ts
ctx.shadowBlur = 6 + bars[i] * 12;
```

## Bug 3: 歌词没有逐字符渲染

**文件**: `apps/web/src/components/KaraokeLyrics.tsx`

**问题**: 当前 active 行只渲染 `{line.content}` 纯文本 + `.karaoke-sweep` CSS 渐变。这不是真正的逐字符渲染——它是一个连续的左到右渐变填充，字符会半亮半暗。

**修复**: 当 `line.words` 存在时（YRC 逐字歌词），将每个字渲染为独立的 `<span>`，用 rAF 直接操作 DOM 切换 `.lit` / `.unlit` 类名。对于没有 word timing 的标准 LRC，保留现有的渐变 sweep 作为 fallback。

**具体实现**:

1. 在 `KaraokeLyrics.tsx` 中，active 行改为渲染字符 spans:
```tsx
<div
  key={`${songId}_${i}`}
  ref={isActive ? activeLineRef : undefined}
  className={className}
>
  {isActive && line.words ? (
    <div className="karaoke-text karaoke-chars" data-line-idx={i}>
      {line.words.map((word, wi) => {
        // Calculate character start times within this word
        const wordStart = word.startMillisecond;
        const wordEnd = line.words![wi + 1]?.startMillisecond 
          ?? lines[idx + 1]?.startMs 
          ?? (wordStart + 3000);
        const charDuration = (wordEnd - wordStart) / word.content.length;
        return Array.from(word.content).map((char, ci) => (
          <span
            key={`${wi}-${ci}`}
            className="karaoke-char unlit"
            data-start={wordStart + ci * charDuration}
          />
        ));
      })}
    </div>
  ) : (
    <div className={`karaoke-text ${isActive ? "karaoke-sweep" : ""}`}>
      {line.content}
    </div>
  )}
  {line.translation && (
    <div className="karaoke-translation">{line.translation}</div>
  )}
</div>
```

Wait, but this won't work because the `textContent` of each span also needs to be set. Let me think again...

Actually the approach should be:
- Render `<span className="karaoke-char" data-start={...}>{char}</span>` for each character
- The rAF loop reads `data-start` and toggles classes

But we need the actual text content inside each span. So:

```tsx
{isActive && line.words ? (
  <div className="karaoke-text karaoke-chars">
    {line.words.flatMap((word, wi) => {
      const wordStart = word.startMillisecond;
      const wordEnd = line.words![wi + 1]?.startMillisecond 
        ?? lines[i + 1]?.startMs 
        ?? (wordStart + 3000);
      const charDuration = (wordEnd - wordStart) / Math.max(word.content.length, 1);
      return Array.from(word.content).map((char, ci) => (
        <span
          key={`${wi}-${ci}`}
          className="karaoke-char unlit"
          data-start={Math.round(wordStart + ci * charDuration)}
        >
          {char}
        </span>
      ));
    })}
  </div>
) : (
  <div className={`karaoke-text ${isActive ? "karaoke-sweep" : ""}`}>
    {line.content}
  </div>
)}
```

2. Modify the rAF loop to update character spans via direct DOM:
```ts
// In the tick function, after finding active line:
if (activeLineRef.current && idx >= 0) {
  const line = lines[idx];
  if (line.words && line.words.length > 0) {
    // Per-character highlighting via direct DOM
    const chars = activeLineRef.current.querySelectorAll('.karaoke-char');
    chars.forEach((el) => {
      const startMs = Number(el.getAttribute('data-start'));
      if (t >= startMs) {
        el.className = 'karaoke-char lit';
      } else {
        el.className = 'karaoke-char unlit';
      }
    });
    // Also update --progress for CSS fallback
    const progress = calcWordProgress(line.words, t, lines[idx + 1]?.startMs, line.startMs);
    activeLineRef.current.style.setProperty('--progress', `${progress}%`);
  } else {
    // Standard LRC: use gradient sweep
    const nextStart = lines[idx + 1]?.startMs ?? (line.startMs + 5000);
    const duration = nextStart - line.startMs;
    const progress = Math.max(0, Math.min(100, ((t - line.startMs) / duration) * 100));
    activeLineRef.current.style.setProperty('--progress', `${progress}%`);
  }
}
```

## Bug 4: 当前唱到的歌词行颜色太淡，看不清

**文件**: `apps/web/src/styles/global.css`

**问题**: 
- `.karaoke-sweep` 的未点亮部分使用 `var(--text-dim)` 即 `#3a3a4e`，在深色背景上几乎看不见
- active 行的 text-shadow `0 0 20px rgba(94,232,197,0.1)` 太弱
- `.karaoke-char.unlit` 使用 `var(--text-dim)` 也太暗

**修复**:
1. `.karaoke-sweep` 渐变的未点亮部分用更亮的颜色:
```css
.karaoke-sweep {
  color: transparent !important;
  --progress: 0%;
  background: linear-gradient(
    to right,
    var(--mint, #5ee8c5) 0%,
    var(--mint, #5ee8c5) var(--progress, 0%),
    var(--text-secondary) var(--progress, 0%),   /* 改为 text-secondary (#8888a0) */
    var(--text-secondary) 100%
  );
  -webkit-background-clip: text;
  background-clip: text;
  -webkit-text-fill-color: transparent;
  font-weight: 700;
}
```

2. `.karaoke-char.unlit` 用更亮的颜色:
```css
.karaoke-char.unlit {
  color: var(--text-secondary);  /* #8888a0 而不是 #3a3a4e */
}
```

3. `.karaoke-char.lit` 增强发光:
```css
.karaoke-char.lit {
  color: var(--mint);
  text-shadow: 0 0 24px var(--mint-glow), 0 0 48px rgba(94,232,197,0.25);
```
