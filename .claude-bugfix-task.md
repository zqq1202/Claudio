- pnpm monorepo: apps/server (Fastify + TypeScript) + apps/web (React 19 + Vite)

## Bugs to Fix (ordered by priority)

### BUG-1: Audio playback fails in browser (CRITICAL)
- **Symptom**: In the browser, ALL /api/audio requests return status 0 (aborted). Via curl `http://localhost:8080/api/audio?id=191254&title=天下&artist=张杰` returns 200 with valid FLAC audio.
- **Root cause investigation needed**: Status 0 in browser typically means CORS, request timeout, or the browser aborting the fetch. Check:
  1. Is there a CORS header missing on the audio endpoint? (Backend may not set `Access-Control-Allow-Origin`)
  2. Does the frontend use `fetch()` or `new Audio()`? Check how the audio player component makes the request
  3. Look at `apps/server/src/routes/` for the audio proxy route handler — does it set proper headers?
  4. Check `apps/web/src/audio/` for the AudioPlayer implementation
- **Files**: `apps/server/src/routes/audio.ts` (or similar), `apps/web/src/audio/`, `apps/web/src/stores/playerStore.ts`

### BUG-2: Play button does not toggle to pause (HIGH)
- **Symptom**: Clicking ▶ does not change to ⏸. Player status stays "待命".
- **Likely related to BUG-1** since playback never starts, the UI never transitions to "playing" state.
- **Investigate**: Even without audio, the play/pause toggle should work at the UI level. Check if the play button's onClick handler correctly dispatches the play action and toggles the local state regardless of audio status.
- **File**: Look for the play button component in `apps/web/src/pages/PlayerPage.tsx` or `apps/web/src/components/`

### BUG-3: Search results missing metadata (HIGH)
- **Symptom**: Backend `/api/search?q=test` returns items with empty artist, album, coverUrl, durationMs. But the raw NCM proxy at port 3000 returns full metadata.
- **Root cause**: The backend search route handler is not forwarding all fields from the NCM response.
- **Fix**: Check `apps/server/src/routes/search.ts` (or wherever search is handled) and make sure it maps artist, album, coverUrl, durationMs from the NCM proxy response.
- **Verify**: NCM returns `result.songs[].artist` as a string — check if the backend tries to parse it differently.
