# Task: Phase 2 — UX 改进 + Phase 3 部分

## 背景
Claudio 是一个 AI 音乐电台播放器。前端 React 19 + Vite + Zustand，后端 Fastify + TypeScript。
暗色主题，glassmorphism 风格。CSS 变量在 apps/web/src/styles/global.css 中定义。

## 任务 1: Toast 通知系统
- 创建 `apps/web/src/stores/toastStore.ts`：Zustand store，`toasts[]`, `addToast(msg, type, duration?)`, `removeToast(id)`
- 创建 `apps/web/src/components/Toast.tsx`：渲染 toast 列表，右上角堆叠，自动消失（默认 3s）
- 在 `apps/web/src/App.tsx` 底部挂载 `<ToastContainer />`
- 添加 CSS：glassmorphism 风格，success(绿)/error(红)/info(蓝)，slide-in-right 动画
- 在以下场景触发 toast：
  - 歌单保存成功 → success
  - 播放错误 → error
  - 设置已保存 → success
  - 搜索无结果 → info

## 任务 2: 键盘快捷键
- 创建 `apps/web/src/hooks/useKeyboard.ts`：
  - `Space` → togglePlay（如果焦点在 input/textarea 则忽略）
  - `ArrowLeft/Right` → seek ±5s
  - `ArrowUp/Down` → volume ±5%
  - `m` → toggleMute
  - `n` → next
  - `p` → previous
- 在 `apps/web/src/App.tsx` 中调用 `useKeyboard()`
- 快捷键触发时在页面底部显示一个简短的快捷键提示（如 "⏸ Paused"），1 秒后消失

## 任务 3: MiniPlayer 集成
- `apps/web/src/components/MiniPlayer.tsx` 已存在
- 在 `apps/web/src/App.tsx` 中：当有 nowPlaying 且不在首页 (location.pathname !== "/") 时，在底部固定显示 `<MiniPlayer />`
- 确保 MiniPlayer 不遮挡内容（main-content 底部加 padding）
- MiniPlayer 需要从 playerStore 读取数据（它已经这样做了）

## 任务 4: 加载骨架屏
- 创建 `apps/web/src/components/Skeleton.tsx`：通用骨架屏组件
- 在 `apps/web/src/pages/PlayerPage.tsx`：当 nowPlaying 为 null 且不是首次加载时显示骨架屏
- 在 `apps/web/src/pages/PlaylistPage.tsx`：歌单列表加载时显示骨架屏
- CSS：脉冲动画，圆角矩形占位

## 任务 5: 响应式优化
- 在 global.css 末尾添加 `@media (max-width: 768px)` 媒体查询：
  - 顶部导航：nav-links 隐藏，只显示 logo 和操作按钮
  - 播放器卡片：全宽
  - 歌曲卡片：单列
  - MiniPlayer：全宽
  - 搜索面板：全宽
  - 聊天区域：适应窄屏
