# SpectrumBars 白屏修复

## 问题
播放歌曲时 SpectrumBars 组件报错导致整个 React 应用白屏。

## 根因
1. `SpectrumBars.tsx` 的 `setupAudio()` 调用 `audioCtx.createMediaElementSource(audioEl)`
2. `AudioPlayer.ts` 的 `ensureGainNode()` 也调用 `this.audioCtx.createMediaElementSource(this.audio)`
3. 浏览器不允许同一个 HTMLAudioElement 创建两个 MediaElementAudioSourceNode，第二个调用抛 InvalidStateError
4. 没有 ErrorBoundary 包裹，组件报错导致整个 React 树崩溃白屏

## 修复方案

### 方案 A: SpectrumBars 复用 AudioPlayer 的 AudioContext
在 AudioPlayerManager 上暴露 AudioContext 和 sourceNode，SpectrumBars 直接复用而不是自己创建。

修改 `apps/web/src/audio/AudioPlayer.ts`:
- 暴露 `getAudioContext()` 方法，返回 AudioContext（lazy init）
- 暴露 `getSourceNode()` 方法，返回 MediaElementAudioSourceNode

修改 `apps/web/src/components/SpectrumBars.tsx`:
- `setupAudio()` 改为从 `audioPlayer.getAudioContext()` 获取已有的 AudioContext
- 从 `audioPlayer.getSourceNode()` 获取已有的 sourceNode
- 自己只创建 analyser 并连接到 sourceNode

### 方案 B: SpectrumBars 的 setupAudio 加 try-catch + App 添加 ErrorBoundary

1. 在 `SpectrumBars.tsx` 的 `setupAudio()` 中加 try-catch 包裹 `createMediaElementSource`，如果失败则静默忽略（频谱只是视觉效果）
2. 在 `apps/web/src/App.tsx` 中用 React ErrorBoundary 包裹 Router，白屏时显示 "加载出错，点击刷新" 的 fallback UI

两个方案都做，A 解决根因，B 是兜底。

## 验证
修改后: `pnpm --filter @ai-radio/web build` 必须通过
然后在浏览器中播放歌曲，确认:
1. 频谱动画正常显示
2. 不再白屏
3. F12 控制台无 React 错误
