**问题**: AI对话面板中推荐的歌曲卡片没有封面图片。

**文件**: `apps/web/src/stores/chatStore.ts`

**修复**: 在所有 `coverUrl` 赋值处确保有 fallback:
```ts
coverUrl: song.cover || `/api/cover?url=${encodeURIComponent(song.cover)}` || `/api/song/cover?id=${songId}`
```

检查所有 `coverUrl` 的赋值点，确保没有空值情况。如果 `song.cover` 为空字符串或 undefined，应该用 `/api/song/cover?id=${songId}` 作为 fallback。

## Bug 2: 底部音乐频谱条/波形条抖动不自然

**问题**: `WaveformBar.tsx` 中的波形条完全静态，不随音乐律动。bars 高度在 `useMemo` 中一次性计算后就不再变化，只有 `bass` prop 的简单乘法。看起来像随机柱子，没有有机的律动感。

**文件**: `apps/web/src/components/WaveformBar.tsx`

**修复方案**: 
- 移除静态 useMemo，改为用 CSS animation 或 JS requestAnimationFrame 驱动每个 bar 的高度
- 给每个 bar 添加不同的动画延迟和频率，模拟音频频谱效果
- 使用 CSS `@keyframes` 让 bars 有机地上下起伏：
  - 每个 bar 用不同的 `animation-duration`（0.5s-1.5s）和 `animation-delay`
  - 使用 `ease-in-out` 缓动函数
  - 当 `isPlaying` 时激活动画，暂停时停止
  - 保持 `bass` prop 的响应性 — bass 值高时振幅更大
- 可以参考常见的 CSS 音频均衡器动画效果

**当前代码**:
```tsx
const bars = useMemo(
  () => Array.from({ length: barCount }, (_, i) => {
    const center = barCount / 2;
    const dist = Math.abs(i - center) / center;
    const base = 4 + (1 - dist) * 20;
    const jitter = Math.random() * 8;
    return base + jitter;
  }),
  [barCount]
);
```
这段只计算一次，需要改成持续动画。

## Bug 3: 歌词需要逐字符渲染（karaoke 效果）

**问题**: 当前歌词显示只做了两件事：
1. LRC 格式歌词：用 CSS gradient sweep（渐变扫描），看起来是整行变色
2. YRC 格式歌词：逐字符高亮（已实现代码）

用户要求**所有歌词都要逐字符渲染**效果，即使是 LRC（只有行时间戳）的歌词也要做逐字效果。

**文件**: `apps/web/src/components/KaraokeLyrics.tsx`

**修复方案**:
- 对于 LRC 格式的歌词（`line.words` 为 null），将每个字符拆分为独立的 `<span>`
- 根据行时长（`nextLine.startMs - currentLine.startMs`）均匀分配每个字符的时间
- 计算方式：`charStartMs = line.startMs + (charIndex / totalChars) * lineDuration`
- 使用与 YRC 相同的 `.karaoke-char.lit/.unlit` CSS class 进行高亮
- 在 rAF loop 中，对 LRC 格式的歌词也执行 per-character DOM 更新

**具体修改**:
在渲染部分，把 LRC fallback 从：
```tsx
<div className={`karaoke-text ${isActive ? "karaoke-sweep" : ""}`}>
  {line.content}
</div>
```
改为也拆字符：
```tsx
{isActive ? (
  <div className="karaoke-text karaoke-chars">
    {Array.from(line.content).map((char, ci) => {
      const totalChars = line.content.length;
      const lineDuration = (nextLineStartMs - line.startMs) || 5000;
      const charStartMs = line.startMs + (ci / totalChars) * lineDuration;
      return (
        <span key={ci} className="karaoke-char unlit" data-start={Math.round(charStartMs)}>
          {char}
        </span>
      );
    })}
  </div>
) : (
  <div className="karaoke-text">{line.content}</div>
)}
```

并且在 rAF tick 中，对 LRC 格式歌词也查询 `.karaoke-char` 并切换 `.lit` class。
