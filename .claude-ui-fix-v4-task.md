# Claudio UI v4 修复任务

## Bug 1: 歌词多行同时变色（应逐字符渲染）

### 根本原因
`.karaoke-sweep` 的 `--progress` CSS 变量设置了 `inherits: true`，但切换 active 行后，旧 DOM 元素（变成 `.past`）保留了 `--progress: 100%`，导致 past 行仍然显示高亮色。应该只让当前 active 行有 sweep 效果，past 行直接显示完成状态，future 行完全不亮。

### 修复方案
**文件: `apps/web/src/styles/global.css`**

在 `.karaoke-sweep` 规则之后添加：

```css
/* Past lines: fully colored, dimmed */
.karaoke-line-wrapper.past .karaoke-sweep {
  --progress: 100% !important;
}

/* Future lines: fully unlit */
.karaoke-line-wrapper.future .karaoke-sweep {
  --progress: 0% !important;
}
```

**文件: `apps/web/src/components/KaraokeLyrics.tsx`**

在 rAF loop 的 `setActiveIndex(idx)` 前，重置上一行的 --progress：

```typescript
// 在 if (idx !== prevIdxRef.current) 块内，setActiveIndex 之前：
if (prevDomRef.current && prevDomRef.current !== activeLineRef.current) {
  const sweep = prevDomRef.current.querySelector('.karaoke-sweep');
  if (sweep) {
    // Past line: set to 100% (fully colored)
    prevDomRef.current.style.setProperty('--progress', '100%');
  }
}
prevDomRef.current = activeLineRef.current;
```

需要添加 `prevDomRef` ref：
```typescript
const prevDomRef = useRef<HTMLDivElement | null>(null);
```

## Bug 2: AI对话封面不显示

### 根本原因
1. Claude 不知道 NCM 封面 URL，返回空 `cover`
2. dispatch.ts 的 NCM search 做了补充，但搜索失败时 cover 仍为空
3. 前端 chatStore.ts 没有 fallback 到 `/api/song/cover?id=`

### 修复方案
**文件: `apps/web/src/stores/chatStore.ts`**

找到 `structuredReply.play` 转换为 `RecommendedSong` 的代码，修改 coverUrl：
```typescript
coverUrl: s.cover || `/api/song/cover?id=${encodeURIComponent(s.id)}`,
```

确保即使 `s.cover` 为空，也会 fallback 到 `/api/song/cover?id=` 端点。

**文件: `apps/server/src/routes/dispatch.ts`**

在 NCM search 补充封面的逻辑中，确保搜索失败时也不会崩溃，且 song.cover 会被正确设置或留空让前端 fallback。

### 根本原因

### 修复方案
**文件: `apps/web/src/styles/global.css`**

1. 更新 CSS 变量：
```css
--bg-card: rgba(26, 26, 38, 0.65);
--bg-card-upper: rgba(18, 18, 28, 0.5);
```

2. 更新 `.player-card`：
```css
.player-card {
  /* ... existing ... */
  background: var(--bg-card);
  backdrop-filter: blur(24px) saturate(1.2);
  -webkit-backdrop-filter: blur(24px) saturate(1.2);

  box-shadow:
    var(--shadow-card),
    0 0 30px rgba(94, 232, 197, 0.08),
    inset 0 0 30px rgba(94, 232, 197, 0.03);

```

3. 更新 `.player-upper`：
```css
.player-upper {
```

4. 更新侧边栏：
```css

  background: rgba(26, 26, 38, 0.75);
```

## Bug 4: Fish Audio TTS 音色不对

### 根本原因
1. **Fish Audio API 直连超时** — `api.fish.audio` 在国内无法直连，需要走 HTTP 代理 (`http://127.0.0.1:7897`)
2. **缺少必要参数** — 没传 `format`、`mp3_bitrate`、`normalize`、`latency`
3. **timeout 太短** — 10s 不够，需要 30s

### 修复方案
**文件: `apps/server/src/services/tts.service.ts`**

需要给 fetch 加上 proxy 支持。Node.js 原生 fetch 不支持代理，需要安装 `undici` 或 `https-proxy-agent`。最简单的方式是用环境变量 `HTTPS_PROXY` + `undici` 的 `ProxyAgent`。

方案A（推荐）: 使用 `undici` 的 ProxyAgent（Node 18+ 内置 undici）：
```typescript
import { ProxyAgent, fetch as undiciFetch } from 'undici';

// 在 synthesize 方法中：
const proxyUrl = process.env.HTTPS_PROXY || process.env.https_proxy;
const fetchOptions: RequestInit = {
  method: "POST",
  headers: { ... },
  body: JSON.stringify(body),
  signal: controller.signal,
};

if (proxyUrl) {
  (fetchOptions as any).dispatcher = new ProxyAgent(proxyUrl);
}

const res = await (proxyUrl ? undiciFetch : fetch)(url, fetchOptions);
```

同时添加参数：
```typescript
const body: Record<string, unknown> = {
  text,
  reference_id: this.voiceId,
  format: "mp3",
  mp3_bitrate: 128,
  normalize: true,
  latency: "normal",
};
```

增加 timeout 到 30s：
```typescript
const timer = setTimeout(() => controller.abort(), 30000);
```

在 `.env` 中确认有代理配置，或在代码中硬编码 WSL 代理地址。
